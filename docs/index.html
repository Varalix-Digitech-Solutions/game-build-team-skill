The gates

"Done" is a loop condition, not an opinion.

The build loop is a deterministic Workflow program — the gates are while-loop conditions in code. A feature that fails either gate goes back with a specific, reproducible punch list. There is no "looks fine to me."

brief → build logic → juice pass → 🧪 TEST GATE → 🎨 CREATIVE GATE → fix → (repeat) → done

🧪 Test gate — rapid

Headless GDScript suite must exit 0, every acceptance criterion asserted on the real call path, a fresh rendered frame checked against the design contract. Runs on a fast simulation — seconds per round, so fix iterations stay cheap.

🎨 Creative gate — fun

Once correct, the running feature is judged against its own brief: readable, responsive, satisfying, on-theme, fair. A dead tap or a silent failure is a defect, even with green tests.

📱 On-device final pass

The Manager builds, installs, and launches the fresh APK on real hardware — confirming the features work together and feel right in-hand. Once, at the end, instead of minutes per round.

The engine

Deterministic order. Dynamic scale. Survives anything.

Built on Claude Code's dynamic Workflow engine — the step order is code, the fleet size adapts to your machine, and the whole run is durable.

  • Resumable across cutoffs. Durable state is written the moment each feature clears a gate. A crash, a pause, or a usage-limit cutoff resumes without redoing approved work.
  • Usage watchdog. A zero-token poller watches your 5-hour usage window; near the cap, agents wind down gracefully and write handoff reports instead of dying mid-thought.
  • Capacity-sized waves. Parallel builders are sized to your host's free memory — a constrained machine never gets OOM-killed.
  • 49 companion skills, no fallback. Godot domain depth (GodotPrompter) plus Karpathy's LLM coding-discipline guidelines, installed project-local. A missing skill stops the run loudly — never a silent downgrade.
  • Discipline built in. Developers state assumptions explicitly, make surgical changes, and never gold-plate; the Tester rejects overcomplicated or out-of-scope diffs even when tests are green.

⚠️ Bring a real spec — the skill verifies, it doesn't guess.

The pipeline's entire value is checking work against a contract, so output quality is capped by the spec you put in. Spend ten minutes with extended thinking ("ultrathink") to pressure-test the idea — mechanics, edge cases, acceptance criteria — and write it down. Then invoke.
